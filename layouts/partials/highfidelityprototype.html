High-Fidelity Prototype

Screen Ideation

The high-fidelity prototype went through several changes after two rounds of user interviews. There were several takeaways that were implemented into the final prototype design.

Dashboard

Multiple changes were made to the dashboard of the app.

  • I blurred the overlay on top of the images to make the text more legible for the WebAim accessibility standards. 
  • The floating action button was removed because users didn’t understand its function on the dashboard. Instead, the function of adding tasks was moved to the individual task cards.
  • The arrows on each task card were changed to “+’s” in order to indicate that the button would have an additive action.
