Low-Fidelity Prototype

Using the site maps and the first user flow, I developed a low-fidelity prototype to present to my team. I created the prototype using a grayscale scheme so that we could focus on the layout instead of the color, something we would incorporate later. 

I took inspiration from the websites that I used in the competitive analysis. Despite being reviewed specifically for an API page, I was still able to use the information I synthesized from the analysis for our Homepage and a Data Insights page. I created them in desktop, tablet, and mobile formats. 

We had not decided on a consistent theme for imagery yet, so I put placeholders where all the images could be included in future iterations.

Desktop Homepage

I took direct inspiration from Notion’s homepage. I incorporated a sample of data and tried to simplify the information on the homepage as much as possible. I wanted to prevent cognitive overload and make navigation as straightforward as possible. 

Because Media is being deemphasized, I moved it to the bottom of the page. The information hierarchy begins with the Insights and Contact Us CTAs, and ends with the Media.

Desktop Data Insights Page

This page is going to be an information hub for potential customers. It will give them samples of data that they can download to have an example of the services Neustreet can provide. This page is geared towards people who are not as tech-savvy, but will likely be on some sort of salesforce team for larger corporations. 

Similar to the Pokemontcg.io website, I divided the Data Insights page into two columns. In this example, I’m using text on the left and an image to illustrate these concepts on the right. These images are placeholders as well, and future images still need to be discussed.

Tablet Homepage

The tablet version of the homepage remains fairly similar to the desktop version, albeit slimmer. There needed to be a negotiation of space in the header to determine whether or not we could fit an image. 

Tablet Data Insights Page

The tablet version of the insights page is virtually identical except for the narrower margins, the hamburger menu, and the account icon at the top. The hamburger menu will be expanded upon at a later date.

Mobile Homepage

The articles and important CTAs were organized into one column for the mobile version of the homepage. I used the data sample as a divider between the content. This is something else that Notion did which I incorporated into our design. 

Mobile Data Insights Page

Just like the mobile homepage, the mobile insights page has been organized into one column. The images have been repositioned to be directly above their corresponding text. The download buttons will be present underneath the text, just like the tablet and desktop versions.
